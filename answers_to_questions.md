# Контрольные вопросы для защиты проекта

## Вопрос 1: Какие стили интеграции использованы и почему?

**Ответ:** 
Использованы два стиля интеграции:
- **REST API (синхронный)** - для назначения курьера, получения статуса, обновления локации. Требуется мгновенный ответ.
- **Асинхронное взаимодействие через базу данных** - модули обмениваются данными через общую SQLite БД.

**Почему:** REST прост в реализации и отладке, а общая БД обеспечивает согласованность данных.

**Ответ:**

Вопрос 8: Как обрабатывается ошибка при недоступности модуля?
Ответ:
Реализованы Retry и Circuit Breaker:
@retry(max_attempts=3, delay=1, backoff=2)
def _make_request(self, method, endpoint, **kwargs)
Circuit Breaker открывается после 3 ошибок на 30 секунд.

Вопрос 15: Разница между синхронным и асинхронным взаимодействием?
Ответ:

Синхронное (REST) - ожидание ответа, блокировка, подходит для операций требующих немедленного результата.

Асинхронное (RabbitMQ) - неблокирующее, очередь сообщений, лучше для длительных операций и высокой нагрузки.

В проекте использован синхронный REST для простоты и наглядности.
