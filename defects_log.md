# Defects Log

## Дефект #1: Курьеры не освобождаются после доставки

| Поле | Значение |
|------|----------|
| **Описание** | После завершения доставки курьер остается в статусе "busy" и не может быть назначен на новые заказы |
| **Серьезность** | Высокая (Critical) |
| **Шаги воспроизведения** | 1. Создать доставку через POST /assign<br>2. Завершить доставку через POST /track/complete<br>3. Проверить статус курьера через GET /couriers |
| **Ожидаемый результат** | Курьер должен стать available = true |
| **Фактический результат** | Курьер остается available = false |
| **Причина** | В методе `complete_delivery` не обновляется статус курьера |
| **Решение** | Добавить `courier.available = True` перед commit |
| **Коммит** | `git commit -m "Fix: Release courier after delivery completion"` |
| **Статус** | ✅ Исправлен |

## Дефект #2: Нет проверки на дублирование task_id

| Поле | Значение |
|------|----------|
| **Описание** | При создании доставки возможно генерация дублирующегося task_id (маловероятно, но возможно) |
| **Серьезность** | Средняя (Medium) |
| **Шаги воспроизведения** | Запустить 1000+ параллельных запросов на создание доставки |
| **Ожидаемый результат** | Все task_id уникальны |
| **Фактический результат** | При коллизии возникает ошибка базы данных |
| **Причина** | Нет повторной попытки при генерации UUID |
| **Решение** | Добавить проверку уникальности и повторную генерацию при коллизии |
| **Коммит** | `git commit -m "Fix: Add uniqueness check for task_id generation"` |
| **Статус** | ✅ Исправлен |

| Поле | Значение |
|------|----------|
| **Описание** | Можно создать доставку с пустым или некорректным номером телефона |

| **Коммит** | `git commit -m "Fix: Add phone number validation"` |
| **Статус** | ✅ Исправлен |

## Дефект #4: Ошибка при попытке завершить несуществующую доставку

| Поле | Значение |
|------|----------|
| **Описание** | При вызове `/track/complete` с несуществующим task_id возвращается 500 ошибка вместо 404 |
| **Серьезность** | Низкая (Low) |
| **Шаги воспроизведения** | POST /track/complete с `task_id: "non-existent-id"` |
| **Ожидаемый результат** | 404 Not Found |
| **Фактический результат** | 500 Internal Server Error (KeyError) |
| **Причина** | Нет проверки существования доставки перед обновлением |
| **Решение** | Добавить проверку `if not delivery: return 404` |
| **Коммит** | `git commit -m "Fix: Return 404 when completing non-existent delivery"` |
| **Статус** | ✅ Исправлен |

## Дефект #5: Нет ограничения на количество повторных попыток при недоступности сервиса

| Поле | Значение |
|------|----------|
| **Описание** | При недоступности одного из модулей, запросы продолжают падать без механизма retry/circuit breaker |
| **Серьезность** | Высокая (Critical) |
| **Шаги воспроизведения** | 1. Остановить tracking модуль<br>2. Вызвать endpoint который вызывает tracking |
| **Ожидаемый результат** | Механизм retry и circuit breaker |
| **Фактический результат** | Сразу падает с ошибкой ConnectionError |
| **Причина** | Отсутствует resilience (retry/circuit breaker) |
| **Решение** | Добавить декоратор @retry и Circuit Breaker |
| **Коммит** | `git commit -m "Add retry mechanism and circuit breaker"` |
| **Статус** | ✅ Исправлен |

## Дефект #6: DeprecationWarning при использовании datetime.utcnow()

| Поле | Значение |
|------|----------|
| **Описание** | В Python 3.14+ `datetime.utcnow()` deprecated |
| **Серьезность** | Низкая (Low) |
| **Шаги воспроизведения** | Запустить любой модуль |
| **Ожидаемый результат** | Нет предупреждений |
| **Фактический результат** | `DeprecationWarning: datetime.datetime.utcnow() is deprecated` |
| **Причина** | Использование устаревшего метода |
| **Решение** | Заменить на `datetime.now(timezone.utc)` |
| **Коммит** | `git commit -m "Fix: Replace deprecated utcnow() with timezone-aware datetime"` |
| **Статус** | ⏳ В ожидании |
